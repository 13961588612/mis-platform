## 1. 产品定义

### 1.1 Product Goals（三目标，彼此正交）

| # | 目标 | 关键结果（可衡量） |
|---|---|---|
| G1 平台化底座 | 建设**一组统一、可复用**的基础设施级 AI 服务，业务模块只依赖契约、不重复造 AI | 基础设施级 AI 服务 ≥ 8 类；新模块对接 agent-gateway 的重复代码量下降 ≥ 80% |
| G2 声明式接入范式 | 定义标准接入约定（SDK/Hook/组件 + 规范），新业务模块**声明式**挂载 AI，接入成本趋近 0 | 接入一个标准 AI 特征的平均工时从 N 天降至 ≤ 0.5 人日；提供接入规范 + 前端 SDK + BFF 代理约定 |
| G3 场景化业务特征 | 将底层能力封装为**可一键挂载的标准业务特征**，覆盖 CRUD/审批/报表/检索/表单/治理主流场景 | 标准业务 AI 特征 ≥ 6 类；后续 ≥ 90% 新业务模块默认挂载至少 1 个 AI 特征 |

> 一句话定位：**AI 不是功能，而是平台默认提供的能力层；业务模块"声明即拥有"。**

### 1.2 User Stories（两类视角）

**A. 平台开发者（内部）**
1. As a 平台开发者，I want 通过统一 AI 接入 SDK/约定声明即可为列表页挂载"自然语言查数 + 智能筛选"，so that 我不必为每个模块单独对接 agent-gateway、处理流式/JWT/脱敏。
2. As a 平台开发者，I want 在表单定义中标记"可由 AI 填充/校验的字段"，so that 新业务表单天然具备 AI 辅助录入，无需额外开发。
3. As a 平台开发者，I want 每个 AI 特征有统一开关与灰度配置，so that 我能在模块上线时按角色/租户控制可见性与范围。

**B. 最终业务用户（各角色）**
4. As a 业务用户（HR/财务/审批人），I want 在任意列表页用自然语言提问就能筛选数据（如"上月未审批的报销单"），so that 我无需记忆复杂筛选条件即可快速定位数据。
5. As a 审批人，I want 审批详情页自动给出 AI 风险摘要（金额/合规点/历史相似案例），so that 我数秒内理解风险，加速且稳健决策。
6. As a 报表查看者，I want 报表旁有 AI 自动解读（趋势/异常/建议），so that 我无须数据分析背景也能读懂报表。

## 2. AI 能力目录（分层）

### 2.1 (a) 基础设施级 AI 服务（平台提供，业务复用）
> 平台统一建设，经 agent-gateway 对外暴露**稳定契约**；业务只依赖契约、不关心模型与实现。

| 服务 | 能力说明 | 复用方（典型业务特征） |
|---|---|---|
| 统一对话/流式 | SSE 流式对话、多模型可插拔（通义/DeepSeek/OpenAI）、上下文管理 | 全局 Copilot、RAG 问答 |
| RAG 检索 | 制度/手册检索 + 引用溯源（向量库 + 重排） | 知识问答、表单填写辅助 |
| NL2SQL | 自然语言→只读 SQL，数据权限沙箱，禁写 | 列表自然语言查数 |
| 内容摘要 | 长文本/多记录摘要 | 详情摘要、审批风险摘要、报表解读 |
| 信息抽取 | 非结构化文本→结构化字段 | 表单自动填充、批量清洗 |
| 分类打标 | 文本分类、实体识别、敏感信息识别 | 审批分类、智能校验 |
| 文本生成 | 模板化生成（通知/报告/话术） | 通知草稿、报表叙述 |
| 智能校验 | 规则 + 模型混合校验（合法性/异常） | 表单校验、数据治理 |

### 2.2 (b) 可嵌入的业务 AI 特征（Feature，标准件）
> 把 (a) 包装为业务模块**一键挂载**的标准特征；每个特征定义"挂载点 + 声明元数据 + 复用底层服务"。

| 业务 AI 特征 | 挂载点（页面位置） | 复用底层服务 | 适用业务场景 |
|---|---|---|---|
| 列表智能筛选 / 自然语言查数 | 列表页右上角 AI 入口 | NL2SQL + 数据权限 | 所有 CRUD 列表 |
| 详情页 AI 摘要 | 详情头部卡片 | 内容摘要 | 单据/档案/客户详情 |
| 表单 AI 自动填充 | 字段右侧"AI 填充"按钮 | 信息抽取 | 录入类表单 |
| 表单 AI 智能校验 | 字段失焦提示 | 智能校验 + 分类打标 | 录入类表单 |
| 审批 AI 风险摘要 | 审批操作区上方横幅 | 摘要 + 抽取 + 分类 | 审批流 |
| 报表 AI 解读 | 图表右上气泡 | 内容摘要 + 文本生成 | 报表/BI |
| 批量数据 AI 清洗 | 导入/治理工具栏 | 信息抽取 + 智能校验 | 数据治理 |
| 全局 Copilot 侧边栏 | 全局抽屉（Cmd+K 唤起） | 全部（上下文感知） | 通用入口 |

## 3. 需求池（P0 / P1 / P2）

> 优先级评判标准：**P0 = 让后续业务模块能低成本接入 AI 的集成范式/SDK/规范**（地基）；P1 = 覆盖核心场景的标准特征；P2 = 增强与运营能力。

### P0（Must have）— 接入范式与底座契约
- **P0-1** 发布《AI 能力接入规范》：明确声明式接入格式、必填/选填元数据、脱敏责任方、鉴权与 traceId 透传约定。
- **P0-2** 前端 AI SDK：`useAI` Hook + `<AiFeature>` 组件族（Copilot/摘要/填充/校验/查数），封装流式、JWT、脱敏、错误态。
- **P0-3** BFF 代理约定：mis-admin-bff 统一代理 agent-gateway（/internal/v1/ai/*），业务前端只调 BFF、不直接连 AI 层。
- **P0-4** 基础设施级 AI 服务**统一契约**：agent-gateway 对外稳定 API（对话/流式、摘要、抽取、NL2SQL、RAG），业务依赖契约不依赖实现。
- **P0-5** 业务 AI 特征"声明式注册表"：feature 清单 + 挂载点约定（列表右上角/表单字段/详情头部），新模块声明即挂载。
- **P0-6** 全局 Copilot 侧边栏（Phase 3 完整流式）+ 页面上下文感知（路由 + 选中行脱敏数据）作为标准入口。

### P1（Should have）— 核心场景标准特征
- **P1-1** 列表自然语言查数 + 智能筛选（NL2SQL 只读沙箱 + 数据权限，结果可存为筛选视图）。
- **P1-2** 详情页 AI 摘要（复用摘要服务，含引用来源）。
- **P1-3** 审批 AI 风险摘要（金额/合规/历史相似/建议，写操作 Human-in-the-loop）。
- **P1-4** 表单 AI 自动填充 / 智能校验（复用抽取 + 校验）。
- **P1-5** RAG 制度/手册问答（带引用溯源）。
- **P1-6** AI 能力统一开关/灰度配置中心（按角色/模块/租户）。

### P2（Nice to have）— 增强与运营
- **P2-1** 报表 AI 解读（图表旁自然语言解读）。
- **P2-2** 批量数据 AI 清洗（导入/治理场景）。
- **P2-3** 对话持久化与历史（多轮上下文留存、跨端）。
- **P2-4** AI 能力使用埋点 + 成本看板（按模块/模型统计 token 与费用）。
- **P2-5** 业务自定义 AI 指令/提示词模板管理。

## 4. UI 设计稿（结构化描述）

> 原则：**全局 Copilot 是通用入口；页面内嵌 AI 特征是场景化快捷件；两者共享同一底层服务与鉴权上下文。**

### 4.1 层级与关系
```
┌─ 全局层 ──────────────────────────────┐
│ 右侧 Copilot 抽屉（Cmd+K / 图标唤起）   │
│ · 自动注入页面上下文（路由+脱敏选中行） │
│ · 发起对话 / 引用页面数据               │
└───────────────┬───────────────────────┘
                │ 共享底层 AI 服务 + 当前用户 JWT
┌─ 页面内嵌 AI 特征（场景化快捷件）───────┐
│ 列表页：右上角 [AI 查数] → 自然语言筛选 │
│ 详情页：头部 [AI 摘要] 卡片             │
│ 表单页：字段 [AI 填充] / 失焦 [AI 校验] │
│ 审批页：上方 [AI 风险摘要] 横幅         │
│ 报表页：图表 [AI 解读] 气泡             │
└────────────────────────────────────────┘
```

### 4.2 各页面 AI 出现方式
- **列表页**：右上角常驻 "AI 查数" 入口 → 弹出自然语言输入框；回车转 NL2SQL 生成筛选条件并叠加到现有列表；结果可"保存为筛选视图"。
- **详情页**：头部固定 "AI 摘要" 卡片，展开显示结构化摘要 + 引用来源（RAG/字段溯源）。
- **表单页**：
  - 字段右侧 "AI 填充" 按钮（基于已有字段 / 上传附件自动抽取填充）。
  - 字段失焦时 "AI 校验" 提示（合法性 / 矛盾检测）。
- **审批页**：操作区上方 "AI 风险摘要" 横幅（金额、合规点、历史相似、建议）；**Human-in-the-loop** 确认后才可执行写操作。
- **报表页**：图表右上 "AI 解读" 气泡，生成趋势 / 异常 / 建议文字。
- **全局 Copilot**：通用对话入口，页面上下文自动注入；可跨页面引用选中行脱敏数据发起追问。

### 4.3 一致性约定
- 所有 AI 入口统一图标 + 触发热键；流式输出统一 loading/skeleton；错误统一降级（AI 不可用时隐藏入口，不影响主流程）。
- 所有 AI 输出默认标注"AI 生成"，关键决策类输出附引用/依据。

## 6. 给架构师的衔接提示（非 PRD 必填，便于落地）
- 复用既有：agent-gateway（AI 层）、mis-gateway（JWT）、mis-admin-bff（代理）、CopilotPanel（已有抽屉）。
- 不新增交易写逻辑到 AI 层；写操作一律 Human-in-the-loop + 回 Java 领域服务。
- 建议优先交付 P0（接入范式/SDK/契约），使后续 P1 特征可"批量化"长出，而非逐个定制。
