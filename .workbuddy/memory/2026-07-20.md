## 修复：子系统左侧面板按钮颜色不正常（灰块 bug）
- 现象：用户反馈子系统左侧面板（侧栏）导航按钮等颜色不正常。
- 根因：侧栏导航项是 `<button class="nav-item">`，但 `.nav-item` 默认态漏设 `background`，浏览器原生灰色按钮底透出，在深墨蓝侧栏上呈现一块块灰块。
- 修复（同文件 `mis-portal-prototype.html`）：
  1. 顶部新增通用重置 `button { background: transparent; border: 0; font: inherit; color: inherit; cursor: pointer; }` 兜底所有裸按钮；
  2. `.nav-item` 显式补 `background: transparent; border: none;`（深墨蓝侧栏透出，无灰块）；
  3. 激活态由「整块实心靛蓝」改为更商务克制的「半透明靛蓝底 `hsl(var(--primary)/.18)` + 左侧 3px 高亮条 `inset box-shadow` + 白字 + 字重 600」。
- 核查：`.search-pill/.avatar/.dropdown-item/.chip/.tab/.quick-card` 均显式设了 background，通用重置无误伤。
- 结论：灰块消失，侧栏恢复「深墨蓝底 + 浅色导航 + 靛蓝高亮激活」的标准企业商务观感。

## 决策：侧栏由深墨蓝改为中灰冷（轻层次框架）
- 用户提问「侧栏也冷灰化是否可以」，经评估给出四选项（保持深蓝/白侧栏+描边/中灰冷侧栏/全冷灰），用户选定**中灰冷侧栏**。
- 理由：全冷灰会退化回「没层次」；中灰冷（`--sidebar: 214 32% 90%` Light / `217 33% 14%` Dark）略深于冷灰画布，靠右侧描边与更浅画布分隔出轻层次，比深蓝轻盈、比全冷灰有框架，统一不抢焦点。
- 同步改动（`mis-portal-prototype.html`）：
  - Light `--sidebar: 214 32% 90%`、`--sidebar-foreground: 222 47% 11%`（深字）、`--sidebar-border: 214 30% 82%`、`--sidebar-accent: 214 32% 84%`、`--sidebar-muted: 215 16% 45%`；
  - Dark 对应：`--sidebar: 217 33% 14%`、`--sidebar-foreground: 210 40% 96%`、`--sidebar-border: 217 33% 22%`、`--sidebar-accent: 217 33% 20%`、`--sidebar-muted: 215 18% 62%`；
  - `.sidebar` 背景由深蓝渐变改为纯色 `hsl(var(--sidebar))`；
  - 激活态 `.nav-item.active` 适配浅底：浅靛蓝底 `hsl(var(--primary)/.14)` + 靛蓝字 `hsl(var(--primary))` + 字重 600 + 左侧 3px 高亮条 `inset box-shadow`（替代之前的深底白字）。
- 文档同步：`portal-design-tokens.md` 的 A.1 表、B.1/B.2 CSS 块、D 节双主题与对比度说明、E 节决策摘要均已更新为「中灰冷」；D 节对比度复核 Light 侧栏前景 #0f1729 vs #dde4ee ≥12:1、激活态靛蓝字 on 浅靛蓝底 ≈5.5:1，均 AA 通过。
- 预览：`mis-portal-prototype.html`（子系统视图，Light/Dark 切换）。

## 修复：每页行数变化仍牵动整页外滚动条（flex basis 泄漏）
- 现象：上一轮滚动已收敛到表格区，但切「每页 50 条」时整页仍出现外滚动条、且随行数变化上下移动。
- 根因：`#list-table-host` 用了 `flex: 1 1 auto`（flex-basis=auto=以内容高度为基准）。表格行数增多 → 该容器 basis 涨到表格自然高度 → 即便 `flex-shrink` 应回收，内容高度仍把外壳撑高、整页变高出现外滚动条；`body` 已有全局 `* { margin:0 }` 重置，故非 margin 问题，而是 flex 内容基准泄漏。
- 修复（`mis-portal-prototype.html`）：
  1. `#list-table-host { flex: 1 1 0%; min-height:0; display:flex; flex-direction:column }`——basis 归零、列向，高度只由剩余空间分配，与表格内容无关，表格再高也只在内部滚动；
  2. `.table-scroll { flex:1 1 0%; min-height:0; overflow:auto; overscroll-behavior:contain }`——阻断滚动链、防滚动手感串到祖先；
  3. `#view-subsystem` 由 `height:100vh` 升级为 `position:fixed; inset:0`（仍 `display:flex;flex-direction:column;overflow:hidden`）——双保险锁死视口，彻底消除整页外滚动条（顺手给 `body` 补显式 `margin:0` 增强可读性，与全局重置一致）。
- 文档同步：`portal-design-tokens.md` E 节 changelog 补 ⑧ 外滚动条修复说明。
- 预览：`mis-portal-prototype.html`（子系统列表页：切每页 10/20/50 条，仅表格内部滚动，整页无外滚动条；Light/Dark）。
