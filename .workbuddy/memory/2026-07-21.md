## 登录页精炼：右侧字号放大 + 整页更商务简洁
- 需求：登录界面右侧登录区域字体稍大一点；整页风格更商务化、简洁化。
- 文件：`docs/frontend/design-proposal/mis-portal-prototype.html`（CSS 252-313 区）+ 令牌文档 changelog ⑨。
- 右侧登录区字号放大（仅在登录卡作用域，避免牵动全局）：
  - 标题 `欢迎登录` 1.5rem→1.75rem；副标 `企业统一管理平台` .875→1rem；
  - 字段标签 `.8125→.875rem`（同步把新建/编辑抽屉的 `.field-label` 也统一到 14px，保持一致）；
  - 登录输入框 `.login-card .input` 与登录主按钮 `.login-card .btn` 设为 `.9375rem` + `min-height:2.75rem`（更大点击区、更稳重）。
- 整页更商务简洁（克制化）：
  - 登录卡内边距 2.25→2.75rem、阴影压平 `0 1px 3px -1px rgb(15 23 42/.07)`（去厚重感）；
  - 表单间距 gap 1→1.125rem、`.login-main` 留白 2.5→3rem，整体更通透；
  - 左侧品牌区副文案 `.95→.9rem`、要点 `.9→.875rem`，使整页更安静；保留大气背景图（不删）。
- 约束：沿用企业靛 shadcn、`--radius:0.375rem` 不变、WCAG AA 对比度不变。
- 预览：`mis-portal-prototype.html`（登录视图，右侧字号明显放大、整页更克制；Light/Dark）。

## 门户首页子系统入口重设计：紧凑瓦片 + 选中视觉
- 需求：门户首页各子系统「风格太松散、空白太多、选中视觉需重设计」。
- 文件：`docs/frontend/design-proposal/mis-portal-prototype.html` + 令牌文档 changelog ⑩。
- 卡片由纵向堆叠大卡 → 横向紧凑瓦片：
  - 栅格 3 列→桌面 **4 列**（`≥1280px`）、间距 `1rem→.75rem`、内边距 `1rem→.75rem .875rem`；
  - 结构改「左圆角方图标 + 右标题/描述单行省略 + 右进入箭头」三栏；新增 `arrowRight` 图标。
- 选中视觉统一语言：左侧 3px 主色高亮条 `::before`（`scaleY` 展开），hover/focus(3px 环)/active 三态一致；hover 叠加主色微染底+抬升+箭头滑入。
- 显式选中态：进入子系统后回填 `.active` 类标记当前所在系统，回门户仍高亮。
- 顶部统计卡间距 `.875rem`、内边距收紧；`.section 2rem→1.5rem`、`.recent/.footer` 同步收敛，消除松散留白。
- 预览：`mis-portal-prototype.html`（门户首页：4 列紧凑瓦片、hover/focus/选中高亮；Light/Dark）。

## 流程记录详情接入门户整体（打通进 mis-portal-prototype.html）
- 需求：「整个页面和集成到门户整体里面去」——把流程记录查看详情从独立文件打通进门户原型。
- 文件：`docs/frontend/design-proposal/mis-portal-prototype.html` + 令牌文档 changelog ⑪；独立 `process-record-template.html` 保留为组件参考。
- 打通方式（复用门户既有，不另起一套）：
  - 流程中心子系统各子菜单进入 → **流程记录列表**（流程编号/名称/发起人/当前节点/状态/发起时间，整行可点+详情按钮），复用门户筛选卡(`col-2`/`col-4`)、状态 Tab(全部/待我审批/我发起的)、分页、表头吸顶。
  - 点行/详情 → **进入详情页**，复用门户 `#sub-content` 容器：`.pr-detail-head`(返回+标题+状态徽标+打印)、`.pr-meta`、`.pr-stepper`(进度)、`.pr-body`(左 Tab[表单/附件/日志]+右信息栏)、`.pr-actions`(底部操作栏)。

  - 详情态隐藏子系统顶 Tab 条、返回列表恢复，列表页码/筛选态保留；`enterSubsystem/activateNav/closeTab` 均重置 `processDetailId` 防卡死。
- 新增样式仅流程专用类 `.pr-*`/`.ro-*`/`.att-*`/`.log-*`/`.aside-*`/`.step-*` + 缺的图标 `PR_ICONS`+`pricon()`；复用门户 `.badge-status/.card/.btn*/.sheet*/.table-scroll/.pager`。Light/Dark 与 AA 不变。
- 校验：node 校验内嵌 JS 语法 OK；grep 确认 `pv-*`/`act-*`/`pane-*` ID 与门户无冲突。
- 预览：`mis-portal-prototype.html`（登录 → 门户首页 → 流程中心 → 待办任务 → 点行进入详情，审批/驳回/撤回演示；Light/Dark）。

## 系统管理控制台（独立模板，14 页完整 CRUD）
- 需求：先查数据库定义文档，设计系统管理各个页面，先确认范围再开始。
- 范围确认（AskUserQuestion）：用户选 **独立模板文件 + 完整 CRUD**，并点名页面集：组织/部门/人员任职职位/APP/API/MODULE/角色权限/菜单/日志/字段。据此定为 14 页（含系统管理既有「用户管理」、审计拆登录/操作日志）。
- 数据库依据：`docs/database/schema-design.md`（system 应用租户控制台表结构）；示例数据参考 `docs/database/seed-data.md`（默认租户/admin/TENANT_ADMIN/system/iam/ops、模块 user/org/system/audit、系统参数等）。平台级 sys_tenant/sys_platform_user 不纳入。
- 新文件：`docs/frontend/design-proposal/system-admin-template.html`（独立高保真模板，复用门户同款企业靛 shadcn 令牌、`--radius:0.375rem`、Light/Dark）。
- 架构（关键）：**数据驱动 CRUD 引擎** + 一个复用 Sheet 抽屉（新建/编辑/详情三态）；各页由 `PAGES[id]` 描述符(filters/columns/form/sample)实例化，新增页只需加描述对象。外壳=顶栏(品牌+租户+主题切换)+左侧分组导航(激活态整行主色)+主区(面包屑/标题/描述/新建)+内容(筛选卡+表头吸顶表格+分页)。
- 14 页映射：组织架构[组织sys_org/部门sys_dept/员工sys_employee/岗位sys_post]、应用与接口[应用sys_app/接口sys_api/模块sys_module]、权限中心[用户sys_user/角色sys_role+权限树+数据范围/菜单sys_menu]、基础数据[字典sys_dict_type/参数sys_config]、审计[登录日志sys_login_log/操作日志sys_oper_log(只读)]。sys_user(每APP账号) 与 sys_employee(自然人) 拆两页。
- 特殊编辑器：角色权限=菜单树复选框(`buildPermTree`，数据 MENU_TREE)+数据范围下拉；菜单管理父级下拉树；详情态角色权限以 `customDetail` 只读摘要呈现。
- 交互/校验：查询/重置、分页(10/20/50+省略号)、行内 详情/编辑/删除、删除确认弹层、租户管理员不可删(防呆)、只读页隐藏新建编辑；必填红框+toast；保存到内存 `dataStore` 即时刷新（纯前端 mock 闭环）。
- 校验：node 语法 OK + DOM 桩跑通 14 页无运行时错误（_smoke.js 临时执行后删除）。
- 文档：`portal-design-tokens.md` 新增 G 节「系统管理控制台（独立模板）」交付说明。
- 预览：`system-admin-template.html`（左侧分组导航切换 14 页；角色页演示权限树+数据范围；日志页只读；Light/Dark）。
