# MIS 管理后台 · UI 设计方案 — 概览

## 产出物
1. **`design-system.md`** — 完整设计系统规格（解决 Brief §8 全部待品牌决策）
2. **`prototype.html`** — 高保真可交互原型（Light/Dark 切换，覆盖 P0+P1 页面）

## 关键决策
- **硬约束遵守**：100% 在 shadcn 令牌体系内；未另起色彩体系、未破坏 `--radius: 0.5rem`、未脱离 shadcn 组件词汇表。用 impeccable 工艺标准（间距节奏 / 视觉层次 / 空态设计）提升质量，但不越令牌红线。
- **用户习惯对齐**：沿用业界后台主流范式——左固定导航 + 顶栏全局操作 + 多 Tab 工作区 + 「筛选栏 + DataTable + 行内操作 + 弹窗」三件套，降低学习成本。
- **已拍板的三项决策（本轮）**：
  - **① 主色 = 企业靛** `#4f46e5`（`243 75% 59%`，白字对比度 ≈6.3:1，AA 通过）；`.dark` 提亮为 `#6366f1`。`--primary` 与 `--ring` 同步改。
  - **② 语义色 = 已采用** 我补的 success / warning / info（HSL 令牌，tint 徽标方案），原型中徽标 / 状态列 / API method 标签均已启用。
  - **③ 登录页 = 统一版 vs 遗留版 并排对比**（见原型登录视图）：统一版走令牌、圆角 0.5rem、中性背景、带描边、跟随主题；遗留版硬编码 #4f46e5、圆角 16px、蓝调渐变、仅亮色。因主色定为靛，颜色债已自然消除，仅圆角/渐变/描边/主题待规范。
- **交互状态补齐**：加载(Skeleton/微 spinner)、空态(插画+引导)、错误态(alert+重试)、无权限静默态(403 占位不弹错)、Toast 语气与位置。

## 覆盖范围（原型）
登录（统一/遗留对比）→ 仪表盘(4 统计卡+快捷入口+Top10 日志) → 用户管理(组织树+表+3 弹窗) → 角色管理(三 Tab Sheet) → 菜单管理(三栏) → 组织/字典/日志占位。全部 Light + Dark，响应式（<1024px 侧栏转抽屉）。
