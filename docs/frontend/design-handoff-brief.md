## 0. 怎么用这份 Brief

1. 设计师的画布 = **已运行的 AppShell 布局壳 + 已装的设计令牌 + shadcn 组件词汇表**。请在这些边界内作画。
2. **不要**脱离 shadcn 词汇表自创基础组件、不要另起一套色彩体系、不要破坏 `--radius` 圆角。否则前端还原必然返工。
3. 输出高保真时，标注：用了哪些 shadcn 组件、间距/圆角取自哪个令牌。

## 2. 设计令牌（已定 vs 待品牌决策）

当前值来自 `src/styles/globals.css`（shadcn 默认蓝，HSL 格式，Tailwind 用 `hsl(var(--x))` 引用）。

| 令牌 | 当前值（light / dark） | 状态 | 品牌需补 |
|------|------------------------|------|----------|
| `--primary` | `221.2 83.2% 53.3%` / `217.2 91.2% 59.8%` | ⚠️ shadcn 默认蓝 | **品牌主色**（如换成企业色） |
| `--radius` | `0.5rem` | ✅ 已定 | — |
| `--sidebar-width` | `16rem`（折叠 `4rem`） | ✅ 已定 | — |
| `--border` / `--input` | `214.3 31.8% 91.4%` / dark `217.2 32.6% 17.5%` | ✅ 已定 | — |
| 字体栈 | `Inter, "PingFang SC", "Microsoft YaHei", sans-serif` | ✅ 已定 | **字阶**（标题/正文/辅助 字号字重） |
| 间距尺度 | Tailwind 默认 4px 基准（`p-4`=16px…） | ✅ 已定 | — |
| 语义色（成功/警告/危险/信息） | 仅 `--destructive` 已定 | ⚠️ 部分 | **补全语义色** |
| 暗色模式配色 | 机制已定（`.dark`），配色待调 | ⚠️ 待定 | 暗色品牌调性 |

> ⚠️ **已知债（如实告知设计师）**：登录页 `login-page.tsx` 仍用遗留 CSS，提交按钮硬编码 `indigo #4f46e5`，与令牌主色（蓝）不一致。设计师若统一视觉，需把登录页也迁到令牌（属后续美化任务，非阻塞）。

## 3. 布局骨架（照此细化，不重画）

```
┌─────────────────────────────────────────────────────┐
│ Header: 面包屑 | Cmd+K | 主题切换 | 用户菜单          │
├──────────┬──────────────────────────────────────────┤
│          │  TabBar（多标签工作区）                    │
│ Sidebar  ├──────────────────────────────────────────┤
│ (可折叠) │  PageHeader（标题 + 操作区）              │
│ NavMenu  │  ┌────────────────────────────────────┐  │
│          │  │ Page Body（页面内容）              │  │
│          │  └────────────────────────────────────┘  │
└──────────┴──────────────────────────────────────────┘
   右侧另有 AI Copilot Sheet 占位（Phase 1 静态）
```

- **顶栏 Header**：面包屑 | Cmd+K 入口 | 主题切换图标 | 用户头像菜单
- **侧栏 Sidebar**：可折叠（`16rem` ↔ `4rem`），递归菜单 `NavMenu`
- **主区**：`PageHeader`（标题+操作按钮区）→ `TabBar`（多标签，类 IDE）→ 内容
- **响应式**：≥1024px 侧栏常驻；<1024px 侧栏变 Sheet 抽屉
- **AI Copilot**：右侧 Sheet 抽屉占位（Phase 1 仅静态欢迎文案 + 禁用输入）

## 5. 组件词汇表（设计师只许用这些）

**shadcn 基础组件（已纳入规划，按需 CLI 添加）：**

```
button, input, label, form, table, dialog, dropdown-menu, select,
checkbox, switch, badge, avatar, separator, sheet, tabs, card,
breadcrumb, pagination, popover, command, skeleton, alert,
tooltip, scroll-area, sidebar
```

**业务组合组件（需设计师给视觉规格）：**

| 组件 | 位置 | 说明 |
|------|------|------|
| `PageHeader` | 每页顶部 | 标题 + 面包屑 + 右侧操作区 |
| `SubmitButton` | 所有保存/删除 | 包装 Button：`loading` 时 `disabled`+`Loader2` 旋转 |
| `DataTable` | 列表页 | 统一表格（排序/分页/选择/空态） |
| `TabBar` | 主区顶部 | 多标签工作区（打开/切换/关闭/固定） |
| `Sidebar` / `NavMenu` | 侧栏 | 可折叠 + 递归菜单 |
| `CopilotPanel` | 右侧 Sheet | AI 占位 |

❌ **禁止**：脱离 shadcn 自创基础组件、新色彩体系、破坏 `--radius` 的圆角。

## 6. 交互与状态设计规范（设计师必须细化）

`admin-web-design.md` 只点名了"空态/错误态"但**未细化视觉**——这部分由设计师补齐：

| 规范 | 来源 | 设计师需补的视觉 |
|------|------|------------------|
| 搜索防抖 300ms | §7.3.5 | loading 态（骨架/微 spinner） |
| 防重复提交：按钮进行中 `disabled`+spinner | §7.3.1/7.3.2 | `SubmitButton` 的 loading 态视觉 |
| **加载态** | — | Skeleton 占位样式 |
| **空态** | — | 空列表插画 + 引导文案 |
| **错误态** | — | 错误占位 + 重试入口 |
| **无权限静默态（403）** | — | 不弹错，显示无感占位（如"无权限查看"空态） |
| **Toast 语气** | sonner | 成功/失败/警告的文案与位置 |

> 关键交互纪律（前端已落实，设计需配合）：搜索/筛选**不实时打接口**（防抖）；写操作按钮**进行中不可再点**。
